# S1 Introduction et révision : programmation orientée objet en C#, travail d'équipe et tests unitaires

## Objectifs
- Revoir la modélisation orientée-objet
- Réviser la manipulation des classes et des objets en C#
- Réviser les tests unitaires
- Pratiquer le travail d'équipe et la gestion de versions
- Aider l'enseignante à ajuster le cours au groupe

## Instructions
- En équipe de 2 ou 3 personnes, réaliser un petit jeu sur la console que vous présenterez au groupe au prochain cours, lundi
- Modéliser la situation. Utiliser les principes de l'orienté-objet (ex. héritage, classes abstraites, interfaces, etc.)
- Limitez vous à quelques classes pour pouvoir présenter lundi prochain sans coder toute la fin de semaine !
- Mettre la modélisation dans le fichier README.md au format mermaid ou sous forme d'image
- Implémenter la solution; écrire des instructions pour démontrer le fonctionnement de votre solution
- Écrire au moins 3 tests unitaires
- Si vous utilisez l'IA, l'indiquer en commentaires pour chaque extrait de code
- Ne pas oublier de remettre le code sur Git

## Ressources suggérées
- Vous pouvez utiliser Visual Studio Live Share (https://visualstudio.microsoft.com/fr/services/live-share/) pour coder dans le même éditeur; je vais l'utiliser avec vous dans les prochaines semaines
- N'hésitez pas à consulter les notes de cours d'applications Windows (https://projets420.gitbook.io/asp.net/420-3p3-dev.-applications-windows)

## Idées si vous manquez d'inspiration
- Jeu de simulation (ex. vie de Bob, jeu de magasin où on veut maximiser des profits avec types d'items, matchs de sports)
- Jeu de combat avec types de monstres différents
